# Google Play Submission Guide

This guide outlines the process for submitting the Currency Converter app to the Google Play Store.

## Phase 1: Preparation

### Prerequisites
- Google Play Console Developer Account ($25 one-time fee)
- Signed AAB (Android App Bundle) file
- App assets (icons, screenshots, feature graphic)
- Privacy Policy URL
- Store listing text

### Required Assets
- App Icon: 512x512 px (PNG)
- Feature Graphic: 1024x500 px (PNG)
- Screenshots: Minimum 2 (Phone), 7-inch tablet (optional), 10-inch tablet (optional)

## Phase 2: Google Play Console Setup

1. Log in to Google Play Console
2. Click Create App
3. Enter App Name: Currency Converter
4. Select Default Language: English (US)
5. Select App (not Game)
6. Select Free
7. Accept Declarations
8. Click Create App

## Phase 3: Store Listing

Navigate to Main Store Listing and fill in the following:

### App Details
- App Name: Currency Converter
- Short Description: Convert 160+ currencies with real-time exchange rates.
- Full Description: (See Store Listing Content)

### Graphics
- Upload App Icon
- Upload Feature Graphic
- Upload Phone Screenshots (2-8 images)

## Phase 4: Data Safety

Navigate to App Content > Data Safety.

1. Data Collection: No
2. Data Sharing: No
3. Submit the form

## Phase 5: Privacy Policy

Navigate to App Content > Privacy Policy.

1. Enter the URL to your hosted Privacy Policy
2. Save

## Phase 6: Content Rating

Navigate to App Content > Content Rating.

1. Start Questionnaire
2. Category: Utility / Productivity
3. Answer No to all questions regarding violence, sexuality, etc.
4. Save and Calculate Rating
5. Apply Rating

## Phase 7: Audience

Navigate to App Content > Target Audience.

1. Select Target Age: 18 and over
2. Appeal to Children: No
3. Save

## Phase 8: Release

Navigate to Production.

1. Click Create New Release
2. Upload the signed AAB file
3. Enter Release Name (e.g., 1.0.0)
4. Enter Release Notes
5. Save
6. Review Release
7. Start Rollout to Production

## Phase 9: Review Process

- Google will review the app (typically 1-3 days)
- Check email for status updates
- Address any rejection reasons if applicable

## Post-Submission

- Monitor crash reports
- Respond to user reviews
- Plan future updates
